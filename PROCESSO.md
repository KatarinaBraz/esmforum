# Processo de Desenvolvimento

## Processo Escolhido

Foi escolhido o modelo Kanban para gerenciamento do projeto.

## Justificativa

O Kanban é adequado para este projeto porque permite uma visualização simples e contínua do fluxo de trabalho. Como as funcionalidades podem ser desenvolvidas independentemente, o modelo facilita a organização, priorização e acompanhamento das tarefas.

Além disso, o Kanban oferece flexibilidade para mudanças durante o desenvolvimento e permite acompanhar o progresso das atividades de forma visual e objetiva.

## Estrutura do Board

O board foi organizado nas seguintes colunas:

- Backlog
- To Do
- In Progress
- Review
- Done

## Funcionalidades Criadas no Board

1. Sistema de votação em perguntas
2. Busca de perguntas por palavra-chave
3. Categorização de perguntas
4. Perfil de usuário
5. Notificações de respostas

## Priorização

As funcionalidades foram priorizadas considerando impacto para o usuário e complexidade de implementação.

Ordem de prioridade:

1. Busca de perguntas por palavra-chave
2. Sistema de votação
3. Categorização de perguntas
4. Perfil de usuário
5. Notificações
