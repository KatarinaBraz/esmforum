# Pair Programming

## Estratégia de Aplicação

O Pair Programming será utilizado durante o desenvolvimento das novas funcionalidades do sistema.

A estratégia adotada consiste em alternar os papéis entre driver e navigator durante as sessões de desenvolvimento.

## Papéis

### Driver

Responsável por escrever o código e implementar as funcionalidades.

### Navigator

Responsável por revisar o código em tempo real, sugerir melhorias, identificar problemas e auxiliar nas decisões técnicas.

## Rotação de Papéis

Os papéis serão alternados periodicamente para garantir participação equilibrada e compartilhamento de conhecimento.

## Ferramentas Utilizadas

As ferramentas planejadas para comunicação e colaboração são:

- VS Code Live Share
- Discord
- GitHub

## Benefícios Esperados

- Redução de erros
- Melhor qualidade do código
- Compartilhamento de conhecimento
- Maior colaboração durante o desenvolvimento
